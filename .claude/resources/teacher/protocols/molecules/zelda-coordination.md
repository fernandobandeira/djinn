# Molecular Protocol: Zelda Coordination
# Structured constraint protocol for seamless knowledge capture orchestration

## Protocol Purpose
This molecular protocol orchestrates atomic constraints to achieve seamless coordination with Zelda during learning sessions, implementing shared constraint spaces for knowledge capture without disrupting learning flow.

## Constraint Architecture Integration

### Atomic Constraints Applied
- **Dialogue Patterns**: Maintain learning flow while enabling capture moments
- **Questioning Syntax**: Use capture opportunities as learning deepening tools
- **Understanding Validation**: Capture readiness as constraint satisfaction evidence
- **Pacing Control**: Coordinate timing to optimize both learning and capture
- **Metacognitive Triggers**: Make knowledge capture process visible and valuable

### Molecular Constraint Orchestration
This protocol coordinates atomic constraints with Zelda's knowledge constraints to create shared constraint spaces where learning validation and knowledge capture reinforce each other without coordination overhead.

## Constraint Coordination Patterns

### High-Priority Coordination (Immediate Delegation)
**Triggers**:
- Major breakthrough with complete understanding validation
- Significant misconception correction with new constraint pattern
- Cross-domain connection with clear transfer evidence
- Boundary insight with limitation recognition

**Protocol**:
- Immediate acknowledgment of insight value
- Quick delegation with rich context
- Continued learning while Zelda processes
- Integration celebration at natural break

### Medium-Priority Coordination (Queued Capture)
**Triggers**:
- Developing insight not yet complete
- Good understanding but still building
- Multiple small insights accumulating
- Session flow too intense for immediate pause

**Protocol**:
- Mental note of capture opportunity
- Continue learning development
- Delegate at natural break or transition
- Batch multiple insights if appropriate

### Low-Priority Coordination (End-of-Session Capture)
**Triggers**:
- Routine application or practice
- Incremental understanding development
- Review or reinforcement activities
- Session energy focused on immediate learning

**Protocol**:
- Note insights for later review
- Focus on immediate learning objectives
- Delegate comprehensive session capture at end
- Plan next session building on captures

## Protocol Adaptation Patterns

### For Breakthrough-Rich Sessions
- Increase coordination frequency
- Use immediate delegation more often
- Celebrate capture moments explicitly
- Build excitement about knowledge creation

### For Exploration Sessions
- Focus on pattern and connection capture
- Use queued coordination for development insights
- Allow extended exploration before capture
- Capture exploration patterns and dead ends

### For Practice Sessions
- Lower coordination frequency
- Focus on application insight capture
- Use end-of-session batch coordination
- Capture what transfers vs. what's contextual

### For Review Sessions
- Coordinate with existing captured knowledge
- Focus on connection strengthening
- Capture evolution of understanding
- Note what persists vs. what fades
