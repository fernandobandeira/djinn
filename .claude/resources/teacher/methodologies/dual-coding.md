# Dual Coding Protocol

## Purpose
Enhance understanding by combining verbal and visual information processing

## When to Use
- Complex relationships
- System architecture
- Process flows
- Abstract concepts
- Memory enhancement

## Process

### Step 1: Verbal Presentation
- Explain in words
- Describe relationships
- Define concepts
- Narrate process

### Step 2: Visual Creation
- Draw diagrams
- Create charts
- Build mind maps
- Sketch representations

### Step 3: Integration
- Connect verbal to visual
- Label diagrams
- Annotate drawings
- Link descriptions

### Step 4: Dual Practice
- Describe from visual
- Draw from description
- Switch between modes
- Reinforce connections

## Verbal-Visual Pairing

### Concept + Icon
- Abstract idea + simple image
- Definition + symbol
- Term + visual cue
- Name + representation

### Process + Flowchart
- Steps in words + visual flow
- Description + diagram
- Narrative + timeline
- Procedure + schematic

### Structure + Diagram
- Hierarchy description + tree
- Relationship text + network
- Organization words + chart
- System description + architecture

### Story + Storyboard
- Scenario text + scene images
- Use case + visual sequence
- Example narrative + illustrations
- Problem story + solution visuals

## Creation Techniques

### Simultaneous Creation
- Explain while drawing
- Narrate the visual
- Build together
- Think aloud

### Sequential Creation
- First explain, then draw
- First sketch, then describe
- Alternate between modes
- Build progressively

### Interactive Creation
- Learner draws while you explain
- You draw while learner explains
- Co-create representation
- Build on each other

## Common Visual Patterns

### For Systems
```
[Component A] --> [Component B]
      |              |
      v              v
[Component C] <-- [Component D]
```

### For Processes
```
Start -> Step 1 -> Decision -> Step 2 -> End
           |          |
           v          v
        Step 1a    Step 2a
```

### For Hierarchies
```
        Root
       /    \
    Node1   Node2
    /  \     |
  Leaf Leaf Leaf
```

### For Relationships
```
   A ---relates---> B
   |                ^
   |                |
   affects      influences
   |                |
   v                |
   C <---impacts----
```
