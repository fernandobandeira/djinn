# Read Together Task

## Purpose
Guide collaborative reading of technical resources with CTO-level analysis

## Process

### 1. Resource Preparation
- Identify resource type (paper, doc, code, article)
- Determine optimal chunking strategy
- Set learning objectives

### 2. Pre-Reading Phase
- Quick scan for structure
- Identify key concepts
- Set expectations
- Ask: "What are you hoping to learn from this?"

### 3. Active Reading Loop
For each chunk:
1. Present content section
2. Highlight critical points
3. Pause for reflection
4. Ask comprehension questions:
   - "What's the core idea here?"
   - "How does this apply to your context?"
   - "What questions does this raise?"
5. Make connections to prior knowledge
6. Explore implications

### 4. Synthesis Phase
- Summarize key takeaways
- Create concept map
- Identify action items
- Document in KB

## Questioning Strategy

### For Technical Concepts
- "How would you implement this?"
- "What are the performance implications?"
- "Where might this break?"

### For Strategic Content
- "What does this mean for your org?"
- "How would you pitch this to your board?"
- "What's the ROI consideration?"

### For Architecture
- "How does this fit your current stack?"
- "What migration path would you take?"
- "What are the team skill implications?"

## Engagement Techniques
- Use analogies to familiar systems
- Create visual representations
- Connect to their specific challenges
- Encourage prediction before revealing
- Celebrate insights and "aha" moments
