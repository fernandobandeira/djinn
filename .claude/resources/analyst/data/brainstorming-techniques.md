# Brainstorming Techniques Reference

## Ideation Techniques

### 1. Brainwriting
**Purpose**: Generate ideas in writing to reduce social pressure
**Process**:
1. Each person writes ideas silently on paper
2. Circulate papers to others
3. Build on existing ideas
4. Continue circulation (3-5 rounds)
5. Compile all ideas

**Benefits**:
- Eliminates groupthink
- Allows introspection
- Introverts participate fully
- Creates written record
- Builds on others' ideas

### 2. Brainstorming (Classic)
**Purpose**: Generate large quantity of ideas in short time
**Process**:
1. Define clear problem statement
2. Set quantity goal
3. Encourage wild ideas
4. Build on others' suggestions
5. Defer all judgment
6. Record all ideas

**Rules**:
- No criticism allowed
- Encourage wild thinking
- Quantity over quality
- Build on others' ideas
- One person talks at a time

### 3. Round Robin
**Purpose**: Structured idea generation with equal participation
**Process**:
1. Sit in circle
2. Each person contributes one idea
3. Pass to next person
4. Skip if no idea ready
5. Continue until exhausted

**Benefits**:
- Equal voice for all
- Prevents dominators
- Structured flow
- Includes quiet voices
- Fair time distribution

### 4. Reverse Brainstorming
**Purpose**: Identify problems by asking opposite questions
**Process**:
1. Rephrase goal as opposite
2. Generate ideas for opposite
3. Reverse each idea back
4. Find solutions

**Example**:
- Original: How to improve customer service?
- Reversed: How to destroy customer service?
- Ideas: Ignore complaints → Listen to complaints
- Long wait times → Reduce wait times

### 5. Silent Brainstorming
**Purpose**: Generate focused ideas without interruption
**Process**:
1. Post prompt visibly
2. Everyone writes individually
3. Share all ideas
4. Group and organize
5. Discuss promising areas

**Benefits**:
- Complete focus
- No interruption
- Captures all voices
- Reduces domination
- Creates documentation

## Expansion Techniques

### 6. SCAMPER
**Purpose**: Prompt idea generation through systematic modification
**Process**:
- **Substitute**: What elements can be replaced?
- **Combine**: What can be merged together?
- **Adapt**: What can be adjusted?
- **Modify**: What can be changed in scale/form?
- **Put to Another Use**: What new uses exist?
- **Eliminate**: What can be removed?
- **Rearrange**: What can be reorganized?

**Application**:
1. List product attributes
2. Ask SCAMPER questions
3. Record all responses
4. Evaluate feasibility
5. Prototype promising ideas

### 7. Morphological Analysis
**Purpose**: Explore systematic combinations of attributes
**Process**:
1. Define problem dimensions
2. List options for each dimension
3. Create matrix of combinations
4. Explore unusual pairings
5. Test combinations

**Example Matrix**:
| Material | Shape | Size | Function |
|----------|-------|------|----------|
| Wood | Round | Small | Store |
| Metal | Square | Large | Protect |
| Plastic | Triangle | Medium | Display |

### 8. Walt Disney Method
**Purpose**: Examine ideas from multiple perspectives
**Process**:
- **Dreamer**: Generate wild ideas
- **Realist**: Plan implementation
- **Critic**: Find problems and improvements

**Workflow**:
1. Role-play as Dreamer (10 min)
2. Switch to Realist (10 min)
3. Switch to Critic (10 min)
4. Return to Dreamer with feedback
5. Iterate until refined

### 9. Attribute Listing
**Purpose**: Generate ideas by examining component attributes
**Process**:
1. List all attributes of subject
2. For each attribute, ask:
   - How can this change?
   - What if it was extreme?
   - What if it was opposite?
3. Combine new attributes
4. Evaluate combinations

**Example**:
- Product: Chair
- Attributes: Legs, seat, back, material, color, height
- Question: What if chair had no legs?
- Ideas: Hammock, bean bag, wall-mounted seating

### 10. Forced Connections
**Purpose**: Create novel ideas by combining unrelated concepts
**Process**:
1. Start with target problem
2. Randomly select object
3. List properties of object
4. Force connections to problem
5. Generate ideas from connections

**Example**:
- Problem: Improve email interface
- Random object: Sponge
- Properties: Absorbent, porous, flexible
- Idea: Make email "absorb" information like sponge
- Concept: Auto-summarize long threads

## Perspective-Shifting Techniques

### 11. Six Thinking Hats
**Purpose**: Examine ideas from six distinct perspectives
**Process**:
- **White Hat**: Facts and information
- **Red Hat**: Emotions and intuition
- **Black Hat**: Critical judgment
- **Yellow Hat**: Optimism and benefits
- **Green Hat**: Creativity and alternatives
- **Blue Hat**: Control and process

**Workflow**:
1. Everyone wears same hat simultaneously
2. 5-10 minutes per hat
3. Rotate through all hats
4. Document insights from each

### 12. Persona-Based Brainstorming
**Purpose**: Generate ideas from different user perspectives
**Process**:
1. Create detailed personas
2. Assign personas to participants
3. Generate ideas from persona viewpoint
4. Record persona-based suggestions
5. Synthesize cross-persona insights

**Persona Elements**:
- Background and goals
- Pain points and frustrations
- Values and priorities
- Technical literacy
- Use case scenarios

### 13. Stakeholder Analysis
**Purpose**: Generate ideas addressing all stakeholder needs
**Process**:
1. Identify all stakeholders
2. Define each stakeholder's needs
3. Brainstorm ideas for each
4. Find overlapping benefits
5. Address conflicting needs

**Stakeholder Categories**:
- Users/customers
- Employees
- Managers/leaders
- Investors
- Partners/suppliers
- Community

### 14. Five Whys
**Purpose**: Generate ideas by exploring root causes
**Process**:
1. State the problem
2. Ask "Why?" - record answer
3. Ask "Why?" about that answer
4. Repeat to level 5
5. Brainstorm solutions at each level

**Example Chain**:
- Why is adoption low?
- Why is onboarding difficult?
- Why haven't we trained users?
- Why do we lack training resources?
- Why wasn't this budgeted?
- Root cause found

### 15. Inverse Brainstorming
**Purpose**: Generate ideas by solving opposite problems
**Process**:
1. Invert problem statement
2. Brainstorm solutions to inverse
3. Reverse each solution
4. Adapt for original problem

**Example**:
- Original: Increase user retention
- Inverse: How to make users leave?
- Ideas: Poor support → Better support
- Complex UX → Simpler UX

## Constraint-Based Techniques

### 16. Constraint Relaxation
**Purpose**: Identify new ideas by removing limitations
**Process**:
1. List all constraints
2. Remove one constraint temporarily
3. Brainstorm with no limit
4. Reintroduce constraint gradually
5. Find creative workarounds

**Constraints to Consider**:
- Technical limitations
- Budget restrictions
- Time constraints
- Resource availability
- Regulatory requirements
- User expertise

### 17. Provocation Technique
**Purpose**: Generate ideas through deliberate provocation
**Process**:
1. Use "PO" (Provocative Operation)
2. State provocative statement
3. Extract ideas from provocation
4. Develop ideas further
5. Evaluate feasibility

**PO Examples**:
- PO: All products should be free
- PO: Customers design the product
- PO: Process takes twice as long
- PO: We eliminate this entirely

### 18. Random Word Technique
**Purpose**: Spark creativity through random associations
**Process**:
1. State problem clearly
2. Select random word (dictionary, spinner)
3. Force associations between word and problem
4. Extract 3-5 ideas
5. Repeat with new words

**Example**:
- Problem: Reduce meeting fatigue
- Random word: Bridge
- Associations: Connect ideas better, span different times, structure
- Ideas: Better agendas, async options, clearer frameworks

## Collaborative Techniques

### 19. Nominal Group Technique
**Purpose**: Combine individual creativity with group discussion
**Process**:
1. Individuals generate ideas silently (10 min)
2. Round-robin sharing (record all)
3. Clarify ideas (no judgment)
4. Group discussion (combine/refine)
5. Vote on best ideas
6. Discuss top vote-getters

**Benefits**:
- Individual time to think
- All ideas recorded
- Equal participation
- Democratic selection
- Documented process

### 20. Affinity Diagramming
**Purpose**: Organize and categorize ideas into themes
**Process**:
1. Brainstorm ideas individually
2. Write each idea on card/sticky
3. Spread cards visibly
4. Group similar ideas (no talking)
5. Create category headers
6. Review and refine groupings
7. Identify major themes

**Benefits**:
- Reveals patterns
- Reduces duplicates
- Organizes thinking
- Finds relationships
- Creates taxonomy
- Aids prioritization

## Selection and Refinement

### Rating Criteria
Use consistent criteria for evaluating brainstormed ideas:
- **Feasibility**: Can we actually do this?
- **Impact**: Will it solve the problem?
- **Cost**: What's the investment?
- **Time**: How long will it take?
- **Risk**: What could go wrong?
- **Innovation**: How novel is this?
- **Alignment**: Does it fit our goals?

### Evaluation Matrix

|------|-------------|--------|------|------|------|-----------|-------|

### Refinement Process
1. Select top 5-10 ideas
2. List pros and cons
3. Identify quick wins
4. Plan implementation
5. Define success metrics
6. Assign ownership
7. Set review schedule

## Best Practices

### Before Brainstorming
- Define clear problem statement
- Set quantity goal
- Identify right participants
- Prepare space and materials
- Establish time limit
- Explain technique clearly

### During Brainstorming
- Encourage wild ideas
- Defer all judgment
- Build on others' contributions
- Record everything
- Keep energy high
- Maintain focus on problem

### After Brainstorming
- Review all ideas
- Group and organize
- Eliminate duplicates
- Evaluate against criteria
- Select for further development
- Communicate results
- Follow up on implementation
