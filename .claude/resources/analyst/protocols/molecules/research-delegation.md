# Research Delegation Protocol

## Core Principles
- Identify research needs systematically
- Prepare comprehensive context for delegation
- Integrate results seamlessly into main workflow
- Maintain quality and coherence standards

## Workflow

### 1. Identify Research Need
- Recognize gaps in current analysis
- Determine research scope and urgency
- Assess feasibility of delegation
- Document specific questions to answer

### 2. Prepare Context
- Summarize current analysis state
- Provide relevant background information
- Define success criteria
- Specify output format expectations

### 3. Delegate to Sub-Agent
- Select appropriate research agent type
- Transmit context and requirements
- Set timeout and quality expectations
- Maintain conversation continuity

### 4. Integrate Results
- Validate result coherence
- Cross-reference with main analysis
- Synthesize into working narrative
- Update knowledge state

## Sub-Agent Types

### Market Research Agent
- Market size and growth analysis
- Customer segmentation insights
- Pricing trend analysis
- Regulatory environment review

### Competitive Intelligence Agent
- Competitor capability mapping
- Market positioning analysis
- Competitive threat assessment
- Strategic differentiation opportunities

### Technical Feasibility Agent
- Technology stack assessment
- Implementation roadmap evaluation
- Risk and dependency analysis
- Resource requirement estimation

### Trend Detection Agent
- Industry trend identification
- Emerging pattern recognition
- Future scenario analysis
- Impact projection

## Integration Strategies

1. **Coherence Verification**
   - Check consistency with existing findings
   - Identify contradictions
   - Request clarification if needed

2. **Synthesis Approach**
   - Blend results with main narrative
   - Highlight novel insights
   - Flag uncertainties

3. **Knowledge Update**
   - Incorporate into working hypothesis
   - Adjust confidence levels
   - Note research dependencies

## Error Handling

- Timeout Management: Return partial results or request summary
- Quality Issues: Request re-research with refined parameters
- Contradictions: Flag for manual resolution or deeper investigation
- Missing Context: Provide additional background and retry
