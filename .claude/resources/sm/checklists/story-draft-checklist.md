# Story Draft Quality Checklist

## Narrative Structure
- [ ] Follows "As a/I want/So that" format
- [ ] Role is clearly defined
- [ ] Benefit is explicit and valuable

## Acceptance Criteria
- [ ] Criteria are numbered
- [ ] Each criterion is measurable
- [ ] Criteria cover different scenarios
- [ ] Align with narrative's intent

## Technical Completeness
- [ ] Dev Notes include architecture references
- [ ] Technical constraints are documented
- [ ] Integration points are clear

## Testability
- [ ] Unit test scenarios are identifiable
- [ ] Integration test needs are specified
- [ ] Acceptance test paths are clear

## INVEST Criteria
- [ ] Independent: Can be developed separately
- [ ] Negotiable: Open to discussion
- [ ] Valuable: Provides clear business value
- [ ] Estimable: Can be reasonably sized
- [ ] Small: Fits within sprint
- [ ] Testable: Can be verified objectively

## Additional Checks
- [ ] No technical jargon preventing understanding
- [ ] Aligns with current sprint goals
- [ ] Potentially blocked dependencies are noted
