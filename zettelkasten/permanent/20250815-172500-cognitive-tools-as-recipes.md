# Cognitive Tools as Recipes

## Core Insight
Cognitive tools provide "recipes" that prevent models from cutting corners. They shape HOW the model thinks, not just WHAT it thinks about. This is constraint on process, not content.

## Evidence & Examples
### Concrete Example
**Bad Recipe**: "Make something delicious" (no process guidance)
**Good Recipe**: "Saut√© onions until translucent, add garlic for 30 seconds, deglaze with wine" (specific process steps)

**Cognitive Tool Example**: 
- Bad: "Analyze this problem" 
- Good: "First identify the root cause, then brainstorm solutions, then evaluate feasibility, then recommend top 3 options"

### Abstract Pattern
Recipes codify successful processes into repeatable procedures that prevent improvisation mistakes.

### Edge Cases
Some situations require improvisation beyond any recipe (novel creative challenges, crisis response).

## Personal Understanding
### My Interpretation
The recipe metaphor explains why cognitive tools work - they encode the "cooking process" that experts use intuitively but models need explicitly.

### Mental Model
Cognitive tools = procedure manuals that prevent shortcuts that seem efficient but produce poor results.

### Confidence Level
High - recipe metaphor clearly explains the mechanism and value of process constraints.

## Open Questions
- How do we identify which steps in a reasoning "recipe" are essential vs optional?
- When does following a recipe become more important than understanding the principles?
- How do we design recipes that teach principles rather than just procedures?

## Application Ideas
- Design cognitive tools as explicit "reasoning recipes" with clear steps
- Create libraries of proven reasoning recipes for different problem types
- Build tools that help extract successful reasoning recipes from expert behavior
