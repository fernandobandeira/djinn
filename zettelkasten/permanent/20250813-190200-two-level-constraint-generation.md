# Two-Level Constraint Generation: Engineered vs. Generated Constraints

## Core Insight
AI systems operate with two distinct levels of constraint generation: engineered constraints (designed and provided externally) and generated constraints (created internally by the model during processing). Memory extraction exemplifies this dual nature - it functions like reasoning where the model generates its own constraint-extraction patterns, revealing that constraint generation itself can be engineered or emergent.

## Evidence & Examples
### Concrete Example
**Level 1 (Engineered)**: "Extract key facts from this text as name-value pairs"
**Level 2 (Generated)**: Model internally develops patterns for identifying what constitutes "key facts" and optimal extraction strategies based on text characteristics

### Abstract Pattern
Constraint generation operates recursively - we can engineer constraints about how constraints should be generated, creating meta-constraint architectures.

### Edge Cases
Some domains may require primarily engineered constraints (high precision), while others benefit from generated constraints (adaptive optimization). The balance point varies by application.
