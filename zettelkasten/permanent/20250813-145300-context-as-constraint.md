# Context as Constraint and Navigation

## Core Insight
Context functions as a constraint system and navigation mechanism rather than an information enhancement tool. It creates a "circle" or "net" around the problem space that limits possibilities and steers the model's attention, acting as boundaries and directional guidance rather than additional content to process.

## Evidence & Examples
### Concrete Example
Instead of providing extensive background about your company, tech stack, and goals (content approach), you constrain with: "I need a Python solution that works with our existing Django REST API and PostgreSQL database" (constraint approach). This creates boundaries that eliminate irrelevant solutions.

### Abstract Pattern
Effective guidance systems limit options rather than expand them - like guardrails on a mountain road that keep you safe by constraining your path, not by adding more road.

### Edge Cases
Over-constraining can eliminate viable creative solutions, requiring balance between constraint and possibility.
