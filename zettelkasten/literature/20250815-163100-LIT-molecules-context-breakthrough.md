## Key Points
### Main Argument
Molecular contexts combine instructions with examples to create "structured constraints around the atom" through few-shot learning. Examples don't teach the model but make desired patterns more probable through pattern recognition.

### Supporting Evidence
1. **Measurable Improvements**: 10-30% accuracy improvement, lower variance, better format adherence
2. **Template Structures**: Prefix-suffix, input-output pairs, chain-of-thought patterns
3. **Example Selection Science**: Diversity, edge cases, ordering, recency bias matter
4. **Diminishing Returns**: Each additional example costs tokens with decreasing benefit
5. **Dynamic Construction**: Context can be optimized per query through similarity matching

### Important Quotes
> "Few-shot learning is when we provide examples of the desired input-output pattern, allowing the model to recognize and continue the pattern."
> — Context: Core mechanism - pattern recognition, not learning

> "The key insight: diminishing returns. Each additional example costs tokens but yields less improvement than the previous one."
