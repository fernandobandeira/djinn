# Shape Up

**Source:** Basecamp (https://basecamp.com/shapeup)

Shape Up is a product development methodology created by Basecamp that emphasizes fixed time, variable scope.

## Core Concepts

### Appetite Over Estimates
Instead of asking "how long will this take?", ask "how much time is this worth?" The appetite (small batch = 1-2 weeks, big batch = 6 weeks) constrains the solution.

### Shaping
Work is "shaped" before being bet on:
- Define the problem and solution at the right level of abstraction
- Remove rabbit holes and edge cases
- Leave room for designers and developers to work out details

### Betting Table
No backlogs. Each cycle, stakeholders review fresh pitches and place bets:
- Pitches not selected are discarded (can be re-pitched later)
- Prevents infinite backlog grooming
- Forces prioritization through scarcity

### Circuit Breaker
If work isn't done when the cycle ends:
- Project stops by default (no extensions)
- Must be re-pitched to continue
- Prevents runaway projects
- Forces scope hammering over timeline extension

### Hill Charts
Track progress by position on a "hill":
- Uphill = figuring things out (uncertainty)
- Downhill = execution (certainty)

## Key Principles

1. **Fixed time, variable scope** - Deadline is sacred, scope flexes
2. **Appetite shapes solution** - Time constraint drives design decisions
3. **No backlogs** - Fresh pitches only, nothing carried forward automatically
4. **Integrated teams** - Designer + developer work together, not handoffs
5. **Maker's schedule** - Uninterrupted time for deep work
