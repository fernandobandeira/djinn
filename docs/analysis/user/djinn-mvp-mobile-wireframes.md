# Djinn MVP Mobile Wireframes - Updated Brand Identity

## Overview
Mobile wireframes for Djinn MVP incorporating the updated brand identity with magic lamp mascot, emerald green color palette, and balanced visual approach (70% solid UI, 20% gradients, 10% liquid glass accents).

## Brand Integration
- **Primary Color**: Emerald Green (#10B981)
- **Mascot**: Magic lamp (teapot-style) with animated liquid interior
- **Style Balance**: Clean white backgrounds with strategic emerald gradients
- **User Context Adaptation**: Professional vs casual user interface variations

## Animation Specifications

### Lamp Mascot States
```css
/* Idle State */
.lamp-idle {
  animation: gentle-sway 3s ease-in-out infinite;
}

@keyframes gentle-sway {
  0%, 100% { transform: rotate(-0.5deg); }
  50% { transform: rotate(0.5deg); }
}

/* Thinking State */
.lamp-thinking {
  animation: liquid-churn 1s ease-in-out infinite;
}

@keyframes liquid-churn {
  0%, 100% { transform: scale(1); }
  50% { transform: scale(1.05); }
}

/* Working State */
.lamp-working {
  animation: active-processing 0.8s ease-in-out infinite;
}

@keyframes active-processing {
  0% { filter: brightness(1); }
  50% { filter: brightness(1.2); }
  100% { filter: brightness(1); }
}

/* Success State */
.lamp-success {
  animation: success-celebration 2s ease-out;
}

@keyframes success-celebration {
  0% { transform: scale(1); }
  20% { transform: scale(1.1); }
  40% { transform: scale(1.05); }
  100% { transform: scale(1); }
}
```

## Responsive Adaptations

### Mobile (320-768px)
- Single column layout
- Simplified lamp animations
- Touch-optimized button sizes (44px minimum)
- Reduced gradient complexity

### Tablet (768-1024px)
- Enhanced lamp animations
- Two-column layout for some screens
- Improved hover states (if touch device supports)

### Desktop (1024px+)
- Full animation suite
- Multi-column layouts
- Rich hover and focus states
- Advanced magical effects

## Accessibility Features

### Visual
- High contrast ratios (4.5:1 minimum)
- Clear focus indicators with emerald outlines
- Scalable text and interface elements
- Color-blind friendly palette

### Motion
- Reduced motion preference respected
- Essential animations only when motion disabled
- Optional magical effect toggles

### Interactive
- Keyboard navigation support
- Screen reader friendly labels
- Touch target minimum 44x44px
- Clear error and success messaging

## User Context Adaptations

### Professional Users (Sarah/Marcus/Robert)
- Minimal lamp animations (subtle breathing/idle only)
- Clean white backgrounds dominate
- Emerald accents limited to CTAs and success states
- Focus on efficiency and task completion
- Reduced magical celebrations

### Casual Users (Alex/Zoe)
- Enhanced lamp personality (more active animations)
- Increased gradient usage (within 20% limit)
- Playful success celebrations
- Richer visual feedback
- Optional magical enhancement toggles

## Implementation Notes

### Performance Considerations
- Optimize gradient rendering for older devices
- Use CSS transforms for animations (GPU acceleration)
- Implement animation frame rate monitoring
- Lazy load complex visual effects

### Development Guidelines
- Use design tokens for consistent color implementation
- Implement proper animation fallbacks
- Test across different screen densities
- Ensure touch target accessibility

### Quality Assurance
- Verify 70/20/10 visual balance across all screens
- Test lamp animation performance on various devices
- Validate accessibility compliance (WCAG AA)
- Confirm brand consistency across user contexts
