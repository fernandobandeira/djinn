## Persona 1: Migration Maya
**The Displaced Mint Power User**

### Demographics
- **Age**: 32
- **Occupation**: Marketing Manager at Tech Company
- **Income**: $75,000/year
- **Location**: Austin, TX
- **Education**: Bachelor's in Communications
- **Family**: Married, no kids yet, planning for future

### Digital Profile
- **Devices**: iPhone 14 Pro, MacBook Pro, iPad
- **Tech Comfort**: High - uses 15+ apps daily
- **Financial Apps**: Previously Mint (5 years), currently testing Monarch & Rocket Money
- **Social Media**: Active on LinkedIn, Instagram for professional networking
- **Banking**: 3 accounts (checking, savings, joint with spouse)

### Financial Behavior
- **Management Style**: Methodical tracker, weekly review sessions
- **Pain Points**: 
  - Lost 5 years of categorized data when Mint shut down
  - Current apps miscategorize 30% of transactions
  - Can't find a single app that does everything Mint did
- **Goals**:
  - Save for house down payment ($50K in 2 years)
  - Track joint expenses with spouse
  - Reduce discretionary spending by 20%
- **Current Workflow**:
  - Sunday morning financial review ritual
  - Manual categorization corrections
  - Excel backup for important data

### Motivations & Frustrations
**Motivations**:
- Data accuracy and historical trends
- Time-saving automation
- Comprehensive financial picture
- Trust and security

**Frustrations**:
- "Why can't any app get my Amazon purchases right?"
- "I miss being able to see everything in one place"
- "Starting over with categories is exhausting"

### Djinn Value Proposition
- **Hook**: Seamless Mint data import with enhanced categorization
- **Wow Moment**: First receipt scan showing item-level detail
- **Premium Trigger**: Need for unlimited accounts and historical data
- **Retention**: Weekly insights that actually save money

### Key Quote
> "I don't need another budgeting app. I need THE budgeting app that actually understands my spending."

### Design Implications
- Prioritize data migration tools
- Focus on categorization accuracy
- Provide comprehensive dashboard views
- Respect existing workflows

## Persona Usage Guidelines

### For Design Decisions
- Always consider at least 2 primary personas for feature evaluation
- Use Maya for data migration and comprehensive features
- Use Tyler for technical and power user features
- Use Sam for engagement and gamification
- Use Dana for detailed tracking and business features

### For Marketing
- Maya: Focus on Mint replacement messaging
- Tyler: Highlight technical capabilities and APIs
- Sam: Emphasize fun and social aspects
- Dana: Stress time-saving and accuracy

### For Testing
- Recruit users matching each persona for usability testing
- Validate assumptions through persona-based interviews
- Track metrics by persona segment
